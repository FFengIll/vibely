# Skill Changelog Directory

This directory contains:
1. **Changelog records** - What changed in each skill version
2. **Version backups** - Complete snapshots of previous skill versions

## Purpose

- **Track skill evolution** - Independent record of each change
- **Enable version recovery** - Restore any previous version
- **Learn from feedback** - Review improvement patterns
- **Maintain quality** - See what worked and what didn't

## Directory Structure

```
.sdlc/changelog/
├── discuss-v1.1.0-20260326.changelog.md    # Changelog: what changed
├── discuss-v1.2.0-20260327.changelog.md
├── coding-v2.0.0-20260328.changelog.md
└── backup/                                  # Version backups
    ├── discuss-v1.0.0-20260326.md.bak      # Original version before v1.1.0
    ├── discuss-v1.1.0-20260327.md.bak      # Before v1.2.0
    └── coding-v1.9.0-20260328.md.bak
```

## File Formats

### Changelog Files

**Naming**: `skillname-v[version]-[date].changelog.md`

**Examples**:
- `discuss-v1.1.0-20260326.changelog.md`
- `coding-v2.0.0-20260327.changelog.md`
- `workflows-feature-v1.2.0-20260328.changelog.md`

### Backup Files

**Naming**: `skillname-v[old-version]-[date].md.bak`

**Examples**:
- `discuss-v1.0.0-20260326.md.bak` (before upgrading to v1.1.0)
- `coding-v1.9.0-20260328.md.bak` (before upgrading to v2.0.0)

**Location**: All backups go in `backup/` subdirectory

## Template

```markdown
# Changelog: [skill-name] v[old] → v[new]

**Date**: 2026-03-26
**Skill**: `[skill-reference]`

## User Feedback Summary
[1-2 sentences max]

## Changes Applied
1. [Change 1]
2. [Change 2]

## Impact
[Expected improvement]
```

## Principles

1. **Concise** - Max 10 lines per changelog
2. **Independent** - Each file stands alone, no cross-references
3. **Actionable** - Focus on what changed and why
4. **Dated** - Always include version and date

## Browsing & Recovery

### View Changelogs

```bash
# View all changes for a skill
ls discuss-*.changelog.md

# View latest changes
ls -t *.changelog.md | head -5

# Search for specific improvements
grep -l "code preview" *.changelog.md
```

### Restore Previous Version

**Manual Recovery:**
```bash
# 1. Find the version you want
ls backup/discuss-*.md.bak

# 2. View the backup to confirm
cat backup/discuss-v1.0.0-20260326.md.bak

# 3. Copy to skill location
cp backup/discuss-v1.0.0-20260326.md.bak actions/discuss.md

# 4. Update version number in the restored file if needed
```

**Agent Recovery:**
```bash
# Ask agent to restore
/sdlc "restore discuss skill to v1.0.0"

# Agent will:
# 1. Find backup/discuss-v1.0.0-*.md.bak
# 2. Identify skill location from backup header
# 3. Copy backup to original location
# 4. Confirm restoration
```

### Version History

To see all versions of a skill:
```bash
# List all backups and changelogs for 'discuss'
ls -t discuss-* backup/discuss-*

# Output:
# discuss-v1.2.0-20260328.changelog.md  (latest changelog)
# discuss-v1.1.0-20260326.changelog.md
# backup/discuss-v1.1.0-20260327.md.bak  (before v1.2.0)
# backup/discuss-v1.0.0-20260326.md.bak  (original)
```
